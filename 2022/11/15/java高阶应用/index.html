1.1
（1）为了表示操作一堆数据而规定的一种统一的标准体系结构–可用于存储多个数据，摆脱了数组存储长度不可变的缺点
（2）java集合框架都对应存储在java.util包下

1.3 list-按照存入数据的顺序，对数据进行存储

（1.3.1）对数组结构的升级，能够实现数组的功能，并且长度可变
（1.3.2）结构特点，存入元素按照存入顺序排列，可以存入相同的数据—（有序可重复）
（1.3.3）实现类-ArrayList/LinkedLIst
#####（1.3.3.1）ArrayList–是一个底层采用了数组结构实现的List接口类(object类型的数组-对象数组)–{transient object[] elemenData}

每次存满时会自己扩容 长度为 { new length = length*1.5} 最大长度限值为:integer-8；(原因在Arraylist最大值限度)
存储的所在数据在内存空间上时连续的
移除中间某个元素效率较低，末尾不受影响
查找元素效率较高

#####(1.3.3.2)/linkdlist
采用双向链表结构，实现的list接口

/*
 * [prev-存储上一个内存地址]--头
 * [item-用于存储数据信息]--身
 * [next-用于存储下一个内存地址]--尾
 *
 */  
  linkdlist 的每一个元素是一个Node<e>对象，存储数据--[没有扩容的概念--理论上可以无限扩容]
  
  结构特征：采用了一个私有的静态内部类组成 
  存储空间不连续：(插入与删除最多只需要三个元素-效率较高)--(也导致查询效率不高)
